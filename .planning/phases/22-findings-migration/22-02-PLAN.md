Add this findings block to both files:

```html
        <div v-if="exhibit.findings?.length" class="exhibit-section">
          <h2>{{ exhibit.findingsHeading || 'Findings' }}</h2>
          <table class="exhibit-table">
            <thead>
              <tr>
                <th>Finding</th>
                <template v-if="exhibit.findings[0].background !== undefined">
                  <th>Background</th>
                  <th>Resolution</th>
                </template>
                <template v-else-if="exhibit.findings[0].severity !== undefined">
                  <th>Severity</th>
                  <th>Description</th>
                </template>
                <template v-else>
                  <th>Description</th>
                </template>
              </tr>
            </thead>
            <tbody>
              <tr v-for="(f, fi) in exhibit.findings" :key="fi">
                <td data-label="Finding">{{ f.finding }}</td>
                <template v-if="f.background !== undefined">
                  <td data-label="Background">{{ f.background }}</td>
                  <td data-label="Resolution">{{ f.resolution }}</td>
                </template>
                <template v-else-if="f.severity !== undefined">
                  <td data-label="Severity">{{ f.severity }}</td>
                  <td data-label="Description">{{ f.description }}</td>
                </template>
                <template v-else>
                  <td data-label="Description">{{ f.description }}</td>
                </template>
              </tr>
            </tbody>
          </table>
        </div>
```

This produces the exact same HTML structure as the old generic table rendering:
- Same `exhibit-table` class
- Same `data-label` attributes on `<td>` elements
- Same column headers matching the original columns arrays
- Same cell values in the same order
- Custom headings rendered via `exhibit.findingsHeading || 'Findings'`

The variant detection uses the presence of `background` field (only Exhibit A) and `severity` field (only Exhibit L). All others default to the Finding/Description 2-column layout.

Do NOT modify the existing generic table rendering -- other non-findings tables still use it.
