## Component Visual Specification

### Personnel Section Wrapper

The personnel section uses the established `.exhibit-section` pattern from existing layout sections:

```
.exhibit-section (personnel wrapper)
  margin-bottom: var(--space-xl)    /* 32px — matches other sections */
  padding-top: var(--space-md)      /* 16px */
  border-top: 1px solid var(--color-border)
```

Source: CONTEXT.md decision "Wrap in `.exhibit-section` div with `<h2>` heading for visual consistency"

### Section Heading

```html
<h2>Project Team</h2>
```

Styled by existing `.page-exhibit-detail .exhibit-section h2`:
- `font-size: var(--font-size-xl)` (24px)
- `margin-top: 0`
- `margin-bottom: var(--space-md)` (16px)

Source: CONTEXT.md decision "Section heading text is 'Project Team'"

### Placement in Layout Template

Personnel section inserts between the sections/resolutionTable block and the `.exhibit-impact-tags` div:

```
sections loop (existing)
resolutionTable (existing)
--- NEW: personnel section ---
exhibit-impact-tags (existing)
```

Source: CONTEXT.md decision "Personnel section appears after the sections loop, before Skills & Technologies"

### Empty State Handling

Guard: `v-if="exhibit.personnel?.length"`

When the guard is false (e.g., Exhibit O with no personnel data):
- No wrapper div rendered
- No heading rendered
- No empty container, no placeholder text

Source: CONTEXT.md decision "Guard with `v-if` -- no wrapper div rendered at all (matches existing empty-section suppression pattern from Phase 16)"

### Responsive Behavior

The personnel grid inside the section inherits Phase 18 responsive behavior:
- Below ~480px: 1 column
- 480px-720px: 2 columns
- 720px+: 3 columns

Mobile section spacing uses existing responsive override: `margin-bottom: var(--space-lg)` (24px) from main.css line 4267.

## Integration Contract

Both layouts receive identical changes. The integration is symmetrical:

| Layout | File | Change |
|--------|------|--------|
| InvestigationReportLayout | `src/components/exhibit/InvestigationReportLayout.vue` | Add import + personnel section |
| EngineeringBriefLayout | `src/components/exhibit/EngineeringBriefLayout.vue` | Add import + personnel section |

### Template Pattern

```html
<!-- After resolutionTable, before exhibit-impact-tags -->
<div v-if="exhibit.personnel?.length" class="exhibit-section">
  <h2>Project Team</h2>
  <PersonnelCard :personnel="exhibit.personnel" />
</div>
```

### Import Pattern

```ts
import PersonnelCard from '@/components/PersonnelCard.vue'
```

Follows existing `TechTags` import pattern already in both layouts.
