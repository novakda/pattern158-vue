```markdown
# Phase 30: Cross-Page Content Audit

**Audited:** {date}
**FAQ source:** src/data/json/faq.json (14 items, 4 categories)
**Comparison pages:** HomePage, TechnologiesPage, PhilosophyPage, ContactPage, CaseFilesPage, exhibit data

## Summary

| Issue Type | Count |
|------------|-------|

### Category: {category heading from faq.ts}

#### Q: "{question text}"

**FAQ Answer (verbatim):**
> {full answer text}

**Cross-Page Comparison:**

| Issue | Type | Source Page | Evidence | Recommended Fix |
|-------|------|------------|----------|-----------------|
| {specific issue} | stale reference / factual drift / content overlap / missing content | {page name} | {what the page says vs what FAQ says} | {concrete fix action} |

If no issues found for a question, include a single row:
| No issues found | — | All pages checked | FAQ content consistent with site | None needed |
```

Classification rules per CONTEXT.md decisions:
- **stale reference**: FAQ uses outdated terminology or links (e.g., "portfolio" instead of "case files")
- **factual drift**: FAQ states facts that contradict current site content (e.g., tech list mismatch)
- **content overlap**: FAQ duplicates content from another page with same audience and no new angle. Same fact reframed for different audience is NOT overlap.
- **missing content**: FAQ omits significant content that other pages highlight (e.g., FAQ tech answer missing React, Python, Power Platform, Claude Code which TechnologiesPage features)

Known issues to verify and document:
1. FAQ Q5 ("legacy systems") references "portfolio" — site uses "case files" since v2.0
2. FAQ Q4 ("technologies") lists only JS/TS/Vue/Node/HTML/CSS/SQL — TechnologiesPage likely includes React, Python, Power Platform, Claude Code
3. FAQ Q6 ("AI and automation") references "GitHub Spec Kit" — verify this term against all site content
4. FAQ Q7 ("industries") lists specific companies — verify against ContactPage and exhibit data
5. FAQ Q8 ("communication") and Q9 ("distributed teams") — check overlap with ContactPage CultureFitSection
6. FAQ Q12 ("typical workflow") — check overlap with PhilosophyPage HowIWorkSection
7. FAQ accessibility mention ("WCAG 2.1 AA") — verify consistent phrasing with ContactPage

Severity ranking is at Claude's discretion. Include a severity column (high/medium/low) if helpful for Phase 31/32 prioritization.
