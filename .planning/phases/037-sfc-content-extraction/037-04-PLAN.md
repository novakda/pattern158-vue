**Structural markup policy (consistent with Plan 02):**
  1. Plain `string` paragraphs — drop inline `<strong>` and `<em>` wrappers.
  2. EXCEPT: the "looking for:" list in RoleFitSection has a genuine `<strong>{role name}</strong> — {description}` structure. Store as `{ name: string; description: string }` and let the template render `<strong>{{ item.name }}</strong> — {{ item.description }}`.

  4. EXCEPT: CultureFitSection has 6 items each with a bolded lead and a paragraph. Store as `{ lead: string; body: string }` and render similarly.
  5. CompensationTable `<table>` structure stays — only the row data moves.
  6. ContactMethods links (email, LinkedIn, GitHub, Case Files) — URLs and labels move to data; the `<a>` and Copy button structural elements stay in the template.

**Why this deviation:** In RoleFit/CompanyFit/CultureFit, the bolded lead is structurally load-bearing (it names the concept being discussed), not decorative emphasis. Dropping it would hurt readability AND lose semantic information that Phase 39 markdown export needs. This is the same principle as preserving the PhilosophyPage blockquote in Plan 02.
