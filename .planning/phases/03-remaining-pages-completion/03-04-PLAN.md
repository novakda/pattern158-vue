    **Component** — `src/components/ExhibitCard.vue`:

    This component demonstrates COMP-04 (named slots for flexible composition):

    ```vue
    <script setup lang="ts">
    import type { Exhibit } from '@/data/exhibits'
    import TechTags from '@/components/TechTags.vue'

    defineProps<{
      exhibit: Exhibit
    }>()
    </script>

    <template>
      <article class="exhibit-card">
        <div class="exhibit-header">
          <span class="exhibit-label">{{ exhibit.label }}</span>
          <span class="exhibit-client">{{ exhibit.client }}</span>
          <span class="exhibit-date">{{ exhibit.date }}</span>
        </div>
        <h3 class="exhibit-title">{{ exhibit.title }}</h3>

        <slot name="quote">
          <!-- Default: render from props -->
          <blockquote class="exhibit-quote">
            <p>{{ exhibit.quote }}</p>
            <footer><cite>{{ exhibit.attribution }}</cite></footer>
          </blockquote>
        </slot>

        <slot name="context">
          <p class="exhibit-context">{{ exhibit.context }}</p>
        </slot>

        <slot name="actions">
          <TechTags :tags="exhibit.impactTags" />
          <router-link v-if="exhibit.exhibitLink" :to="exhibit.exhibitLink" class="exhibit-link">
            View Full Report &rarr;
          </router-link>
        </slot>
      </article>
    </template>
    ```

    Named slots (quote, context, actions) allow consuming templates to override any section while keeping sensible defaults from props. This satisfies COMP-04.
