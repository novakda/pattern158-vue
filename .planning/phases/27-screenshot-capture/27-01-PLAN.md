    **Step 2: Add 'capturing' to FeedbackPhase (per SHOT-02)**
    In `feedback.types.ts`, update the FeedbackPhase union to include 'capturing' between 'picking' and 'annotating':
    ```
    export type FeedbackPhase = 'idle' | 'picking' | 'capturing' | 'annotating' | 'submitting' | 'done' | 'error'

    **Step 3: Create screenshotCapture.ts (per SHOT-01)**
    Create `src/components/feedback/screenshotCapture.ts` that:
    - Exports an async function `screenshotCapture(el: HTMLElement): Promise<string>`
    - Lazy-loads html2canvas via `const { default: html2canvas } = await import('html2canvas')`
    - Calls `html2canvas(el, { scale: 1, logging: false, useCORS: true })` to capture the element
    - Converts the returned canvas to a PNG data URI via `canvas.toDataURL('image/png')`
    - Returns the data URI string
    - Throws on failure (caller handles error)

    **Step 4: Update useFeedback.ts selectElement flow**
    Modify `selectElement` to become async and include the capture step:
    - Rename current `selectElement` logic or refactor it
    - New flow: `selectElement(el, capture)` sets state.selectedElement and state.capture immediately, sets phase to 'capturing'
    - Then calls `screenshotCapture(el)` in a try/catch
    - On success: sets `state.capture.screenshotDataUri = dataUri`, sets phase to 'annotating'
    - On failure: `console.warn('Screenshot capture failed:', error)`, sets `state.error = 'Screenshot capture failed. You can retry or continue without screenshot.'`, sets phase to 'error'
    - Import screenshotCapture from `@/components/feedback/screenshotCapture`

    **Step 5: Add spinner CSS to feedback.css (per SHOT-02)**
    Add these styles to `feedback.css`:

    - `@keyframes fb-spin { to { transform: rotate(360deg) } }`
    - `.fb-capture-label` — color: var(--fb-text), margin-top: 12px, font-size: 14px, text: "Capturing..."

      </div>
    </div>
    ```
