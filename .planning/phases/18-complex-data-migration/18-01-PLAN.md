**Replace 3 data files with thin loaders:**

**src/data/findings.ts:**
```typescript
import type { Finding } from '@/types'
import findingsData from './json/findings.json'

export type { Finding }
export const findings: Finding[] = findingsData as Finding[]
```
Note: `as Finding[]` type assertion needed because JSON data has optional `link` field — TypeScript's inferred JSON type won't match the optional property exactly.

**src/data/philosophyInfluences.ts:**
```typescript
import type { PhilosophyInfluence } from '@/types'
import philosophyInfluencesData from './json/philosophyInfluences.json'

export type { PhilosophyInfluence }
export const philosophyInfluences: PhilosophyInfluence[] = philosophyInfluencesData
```

**src/data/influences.ts:**
```typescript
import type { Influence, InfluenceLink, InfluenceSegment } from '@/types'
import influencesData from './json/influences.json'

export type { Influence, InfluenceLink, InfluenceSegment }
export const influences: Influence[] = influencesData as Influence[]
```
Note: `as Influence[]` type assertion needed because nested optional `link` field in segments causes JSON type mismatch.

Key conventions (from Phase 17):
- Optional keys omitted entirely from JSON when not present (not set to null)
- Unicode chars stored as escaped sequences (\u2014 for em dash, \u2019 for right single quote, \u201c/\u201d for curly double quotes)
- Thin loaders re-export both data const and type for backward compatibility
