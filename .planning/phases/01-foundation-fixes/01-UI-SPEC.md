## Spacing Scale

Declared values — using project's existing `--space-*` CSS custom properties (rem-based, not px-based):

| Token | CSS Custom Property | Approximate px | Usage |
|-------|--------------------|-----------------|--------------------------------------------------------------------|
| xs | `--space-xs` | 4px (0.25rem) | Icon gaps, inline padding |
| sm | `--space-sm` | 8px (0.5rem) | Compact element spacing |
| ms | `--space-ms` | 12px (0.75rem) | Mobile horizontal padding, small inline gaps |
| md | `--space-md` | 16px (1rem) | Default element spacing, body content flow |
| lg | `--space-lg` | 24px (1.5rem) | Section padding, card internal spacing |
| xl | `--space-xl` | 32px (2rem) | Layout gaps between major elements |
| 2xl | `--space-2xl` | 48px (3rem) | Major section breaks |
| 3xl | `--space-3xl` | 64px (4rem) | Page-level vertical rhythm |
| 4xl | `--space-4xl` | 80px (5rem) | Hero/feature section spacing |
| 5xl | `--space-5xl` | 96px (6rem) | Maximum vertical spacing (hero backgrounds) |

Exceptions: none for Phase 1. Phase 1 adds one new component (NotFoundPage) — it uses `--space-lg` and `--space-xl` for container padding, consistent with PhilosophyPage reference. No touch-target exceptions needed (no interactive icons added in this phase).

Source: `src/assets/css/main.css` lines 86–95.

## Typography

Using project's existing `--font-size-*` and `--line-height-*` CSS custom property pairs. Phase 1 introduces only the NotFoundPage component — four type roles required:

| Role | CSS Token | Approx Size | Weight | Line Height Token | Approx Line Height |
|------|-----------|-------------|--------|-------------------|--------------------|
| Body | `--font-size-base` | 16px (1rem) | 400 (regular) | `--line-height-base` | 1.6 |
| Label / UI text | `--font-size-sm` | 14px (0.875rem) | 400 (regular) | `--line-height-sm` | 1.5 |
| Heading (h1) | `--font-size-4xl` | 56px (3.5rem) | 400 (Bebas Neue has no bold variant) | `--line-height-4xl` | 1.2 |
| Subheading / paragraph lead | `--font-size-lg` | 20px (1.25rem) | 400 (regular) | `--line-height-lg` | 1.6 |

Font family assignment:
- Headings (`h1`, `h2`, `h3`): `var(--font-heading)` → `'Bebas Neue', 'Arial Black', sans-serif`
- All other text: `var(--font-body)` → `'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`

Weights: exactly 2 — 400 (regular, all body and label copy) + Bebas Neue is inherently display-weight by typeface design (no numeric weight variant used for headings).

Source: `src/assets/css/main.css` lines 43–83.

## Color

Project uses a dual-mode token system (light + dark via `[data-theme="dark"]`). Both modes must satisfy WCAG AA contrast. Phase 1's NotFoundPage uses only the three primary surface roles and the accent for the CTA link.

| Role | Light Mode Token / Value | Dark Mode Value | Usage |
|------|--------------------------|-----------------|-------|
| Dominant (60%) | `--color-background` `#faf9f6` | `#1a2838` | Page background, base surface |
| Secondary (30%) | `--color-background-alt` `#e8e8e8` / `--color-surface` `#ffffff` | `#243447` / `#2a3f54` | Section alternation, container panels |
| Accent (10%) | `--color-primary` `#0e7c8c` | `#30c9dc` | CTA link/button on NotFoundPage only |
| Destructive | `--color-danger` `#c82333` | `#ff8c99` | Not used in Phase 1 — no destructive actions |

Accent reserved for (Phase 1 scope):
- The "Back to Home" `<RouterLink>` rendered as `btn btn-primary` on NotFoundPage
- No other elements in Phase 1 use accent color

Text colors:
- Body text: `--color-text` `#2d3436` light / `#e8e8e8` dark (12:1+ contrast on background)
- Headings: `--color-heading` `#1a2838` light / `#e8e8e8` dark
- Muted/secondary text: `--color-text-muted` `#666666` light / `#ababab` dark (both WCAG AA)

Source: `src/assets/css/main.css` lines 10–41 (light) and 120–177 (dark).

## Copywriting Contract

Phase 1 introduces one new user-facing component: NotFoundPage. The three existing pages being fixed (TechnologiesPage, ContactPage, HomePage) have no copy changes — only HTML structure changes.

| Element | Copy |

## Interaction Contract

Phase 1 interactions are limited to navigation only:

| Interaction | Element | Behavior |
|-------------|---------|----------|
| 404 navigation | `<RouterLink to="/">` on NotFoundPage | Standard Vue Router push; no custom transition |
| Skip link | `<a href="#main-content">` in App.vue | After fix, resolves to App.vue's `<main id="main-content">` — exactly one element with this id (correct state) |
| Theme toggle | `ThemeToggle.vue` (existing, unchanged) | Applies `[data-theme="dark"]` to root; NotFoundPage inherits token overrides automatically |

No hover states, modals, form interactions, or loading states added in Phase 1.
