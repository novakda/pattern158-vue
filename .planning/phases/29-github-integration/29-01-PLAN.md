Create `src/components/feedback/githubSubmit.ts` as a pure async function (not a composable -- no reactive state).

**Function signature:**
```typescript
export interface SubmitResult {
  issueUrl: string
}

export interface SubmitParams {
  token: string
  repo: string
  labels: string[]
  capture: ElementCapture
  comment: string
  environment: EnvironmentMeta
}

export async function submitFeedback(params: SubmitParams): Promise<SubmitResult>
```

**Gist upload (primary path per D-01 GH-01):**
1. Generate filename: `feedback-screenshot-${Date.now()}.png`
2. POST to `https://api.github.com/gists` with:
   - `public: false` (secret gist)
   - `files: { [filename]: { content: capture.screenshotDataUri } }` (the full data URI string as content)
   - Headers: `Authorization: token ${params.token}`, `Accept: application/vnd.github+json`
3. Extract `raw_url` from response: `response.files[filename].raw_url`
4. If Gist returns 404, console.warn "Token may lack 'gist' scope"
5. Embed in issue body as `![Screenshot](${raw_url})`

**Data URI fallback (per D-03 GH-03):**
If Gist POST fails for ANY reason:
1. console.warn identifying Gist failure reason
2. Start with the existing PNG data URI from `capture.screenshotDataUri`
3. If issue body total would exceed 60,000 chars:
   - Create an offscreen canvas, draw the image, re-encode as JPEG at quality 0.5: `canvas.toDataURL('image/jpeg', 0.5)`
   - If STILL exceeds 60,000 chars: set screenshot string to `"> Screenshot omitted: too large to embed"`
4. Embed the data URI (or omission note) in issue body

**Issue creation (per D-02 GH-02):**
POST to `https://api.github.com/repos/${params.repo}/issues` with:
- `title`: First 80 chars of comment, or "Visual feedback" if comment is short/generic
- `body`: Structured markdown (see template below)
- `labels`: params.labels (per D-04 GH-04)
- Headers: same auth headers as Gist call

**Issue body markdown template:**
```markdown
## Feedback

{comment}

## Screenshot

{screenshot_embed}

## Element

| Field | Value |
|-------|-------|
| Tag | `{capture.tag}` |
| Selector | `{capture.selectorPath}` |
| Bounding Rect | {top}, {left}, {width}x{height} |
| Component | {capture.componentName ?? 'N/A'} |

## Environment

| Field | Value |
|-------|-------|
| URL | {environment.pageUrl} |
| Viewport | {environment.viewport.width}x{environment.viewport.height} |
| User Agent | {environment.userAgent} |
```

**Error handling:**
- Distinguish error types by HTTP status:
  - 401/403: "Authentication failed -- check that your GitHub token is valid and not expired"
  - 403 with `X-RateLimit-Remaining: 0`: "GitHub API rate limit exceeded. Try again in {minutes} minutes"
  - 404 on issue creation: "Repository not found -- check VITE_GITHUB_REPO format (owner/repo)"
  - 422: "Issue creation rejected by GitHub -- body may be too large"
  - Network error (fetch throws): "Network error -- check your internet connection"
- Throw an Error with the descriptive message string (useFeedback will catch and set state.error)

**Do NOT:**
- Import Octokit (native fetch only, per CONTEXT.md decisions)
- Use any reactive Vue state -- this is a pure async function
- Store any state -- caller manages state via useFeedback

3. Add a new `submit()` async function:
   ```typescript
   async function submit() {
     const config = useFeedbackConfig()
     if (!config.isConfigured || !state.capture) return

     state.phase = 'submitting'
     state.error = null

     try {
       const result = await submitFeedback({
         token: config.token,
         repo: config.repo,
         labels: config.labels,
         capture: state.capture,
         comment: state.comment,
         environment: {
           pageUrl: window.location.href,
           viewport: { width: window.innerWidth, height: window.innerHeight },
           userAgent: navigator.userAgent,
         },
       })
       state.issueUrl = result.issueUrl
       state.phase = 'done'
     } catch (err) {
       state.error = err instanceof Error ? err.message : 'Submission failed'
       state.phase = 'error'
     }
   }
   ```
4. Export `submit` from the returned object of useFeedback()
5. Add a `retry()` function that calls `submit()` again (re-attempts with same state data)
