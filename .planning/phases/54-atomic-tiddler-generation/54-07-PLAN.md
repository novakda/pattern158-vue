**Link-syntax regex (LOCKED by CONTEXT.md line 41):** `/\[\[([^\]|]+)(?:\|[^\]]+)?\]\]/g`

- Group 1 = the target title (the link destination)
- Optional `|{display}` segment follows target but does not factor into integrity checking

TiddlyWiki wikitext allows two link forms:
- `[[Home]]` → target = `Home`
- `[[Click here|Home]]` → display = `Click here`, target = `Home`

The CONTEXT-locked regex puts target in group 1 for both forms by consuming `[^\]|]+` up to `|` or `]`. The optional non-capturing `(?:\|[^\]]+)?` swallows the display portion when present. This means:
- `[[Home]]` → group 1 = `'Home'`
- `[[Home|Click here]]` → group 1 = `'Home'`  (target-first form, display after pipe)

Wait — TiddlyWiki's actual convention is `[[display|target]]`, NOT `[[target|display]]`. The CONTEXT regex description says "first group is target", which means the regex defines target as the part BEFORE the pipe. For a `[[display|target]]` actual TiddlyWiki pipe-form link, group 1 would grab the display (wrong). Resolution: CONTEXT.md line 41 LOCKS the regex as written. Phase 54 generators emit ZERO piped links — every `[[...]]` they produce is the unpiped form. So the integrity checker's behavior for piped links is defined but not heavily exercised by the Phase 54 generator set. A piped-form test case is added to the test suite to document the behavior locked by the regex: group 1 is the substring BEFORE `|` (regardless of which actual side is target in TiddlyWiki's semantics). The integrity check is CORRECT for the Phase 54 generators because they only emit unpiped `[[X]]` links.

```typescript
// scripts/tiddlywiki/generators/integrity-check.ts
// Phase 54 Plan 07 — ATOM-05 consumer half.
// Walks tiddler bodies and reports [[target]] links whose target is not a
// tiddler title in the given set.
// SCAF-08 policy: no wall-clock reads, no instantiated dates, no parallel
// iteration helpers, deterministic output.

import type { Tiddler } from './types.ts'

const LINK_REGEX = /\[\[([^\]|]+)(?:\|[^\]]+)?\]\]/g

export interface Orphan {
  readonly source: string
  readonly link: string
}

export interface IntegrityResult {
  readonly orphans: readonly Orphan[]
}

function extractLinkTargets(body: string): string[] {
  const targets: string[] = []
  // String#matchAll is ES2020+, in-spec under tsconfig.scripts.json lib=ES2022.
  for (const match of body.matchAll(LINK_REGEX)) {
    targets.push(match[1])
  }
  return targets
}

function compareOrphan(a: Orphan, b: Orphan): number {
  if (a.source !== b.source) return a.source < b.source ? -1 : 1
  if (a.link !== b.link) return a.link < b.link ? -1 : 1
  return 0
}

export function verifyCrossLinkIntegrity(
  tiddlers: readonly Tiddler[],
): IntegrityResult {
  if (tiddlers.length === 0) return { orphans: [] }
  const titles = new Set<string>()
  for (const t of tiddlers) titles.add(t.title)
  const orphans: Orphan[] = []
  for (const t of tiddlers) {
    const targets = extractLinkTargets(t.text)
    for (const target of targets) {
      if (!titles.has(target)) {
        orphans.push({ source: t.title, link: target })
      }
    }
  }
  orphans.sort(compareOrphan)
  return { orphans }
}
```

Notes:
- `LINK_REGEX` with `g` flag supports `matchAll` iteration. The regex is the EXACT string from CONTEXT.md line 41.
- `matchAll` instead of `.match(regex)` with `g` — `.match` with `/g` returns just the full-match strings (no groups), while `matchAll` returns RegExpMatchArray per match with `[1]` = first capture group. ES2020 feature, allowed by ES2022 lib.
- Each orphan is a `{ source, link }` pair. Same source + same link reported once per occurrence; deduping across the same tiddler is NOT performed (if a body has two `[[Ghost]]` links, two orphans emit). Phase 54 generators do not produce duplicate inline links so this doesn't blow up production output; if it ever does, a Set-based dedupe can be added without changing the public shape.
- `compareOrphan` guarantees deterministic ordering — sort first by source, then by link.
- The function is TOTAL: every input type-checks, nothing throws.
- Empty input short-circuits to `{ orphans: [] }` without entering the loop.
