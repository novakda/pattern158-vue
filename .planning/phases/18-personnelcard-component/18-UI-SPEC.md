## Typography

Only tokens relevant to the PersonnelCard component. All use `--font-body` (Inter).

| Role | Size | CSS Variable | Weight | Line Height |
|------|------|-------------|--------|-------------|
| Name (primary) | 16px (1rem) | `--font-size-base` | 600 (semibold) | 1.6 (`--line-height-base`) |
| Role | 14px (0.875rem) | `--font-size-sm` | 400 (regular) | 1.5 (`--line-height-sm`) |
| Title | 14px (0.875rem) | `--font-size-sm` | 400 (regular) | 1.5 (`--line-height-sm`) |
| Organization | 14px (0.875rem) | `--font-size-sm` | 400 (regular) | 1.5 (`--line-height-sm`) |

**Anonymous name substitute (title in name position):** 16px (`--font-size-base`), weight 400 (regular), italic, color `--color-text-muted` (#666666). This combination signals "role description, not a person's name" without appearing as a warning or error.

## Component Visual Specification

### PersonnelCard Grid Container

```
.personnel-grid
  display: grid
  grid-template-columns: repeat(auto-fill, minmax(240px, 1fr))
  gap: var(--space-md)        /* 16px */
```

Responsive behavior:
- Below ~480px viewport: collapses to 1 column (natural CSS grid behavior)
- 480px-720px: 2 columns
- 720px+: 3 columns (max with current minmax)

Source: CONTEXT.md D-02; RESEARCH.md Pattern 3

### Individual Card

```
.personnel-card
  padding: var(--space-md)          /* 16px */
  border: 1px solid var(--color-border)
  border-radius: var(--radius-md)   /* 4px */
  background: var(--color-surface)
```

### Self-Entry Card Modifier

```
.personnel-card--self
  border-left: 3px solid var(--color-primary)
  background: var(--color-primary-subtle)
```

Source: CONTEXT.md discretion area (self-entry visual treatment); RESEARCH.md Pattern 3

### Card Information Order (top to bottom)

**Named person (has `name` field):**
1. Name -- `--font-size-base`, weight 600, color `--color-heading`
2. Role (if present) -- `--font-size-sm`, weight 400, color `--color-text`
3. Title -- `--font-size-sm`, weight 400, color `--color-text-muted`
4. Organization -- `--font-size-sm`, weight 400, color `--color-text-muted`

**Anonymous person (no `name` field):**
1. Title (in name position) -- `--font-size-base`, weight 400, italic, color `--color-text-muted`
2. Role (if present) -- `--font-size-sm`, weight 400, color `--color-text`
3. Organization (if present) -- `--font-size-sm`, weight 400, color `--color-text-muted`

Source: CONTEXT.md D-03 (role > title priority), D-04 (title as name substitute), D-06 (muted/italic), D-07 (render whatever exists)

### Field Spacing Within Card

Each field line separated by `--space-xs` (4px) margin-bottom. No dividers or separators between fields.

### Self-Entry Positioning

Maintain original data order. Self entries do not sort to the top.

Source: Discretion decision based on data inspection -- each exhibit has 1 self entry among 2-12 total. Reordering would be disorienting when cross-referencing with the original table section.

### Section Heading

Component does NOT render a section heading. Heading responsibility deferred to Phase 19 layout integration, consistent with how TechTags and other child components work.

Source: RESEARCH.md Section Heading Pattern Analysis

### Large List Handling

No collapsible threshold. Maximum 12 entries (Exhibit A) fits comfortably in a 3-column grid without scrolling.

Source: RESEARCH.md Data Inventory (max 12 entries)

## Display Mode Decision Table

| Condition | CSS Class | Visual Treatment |
|-----------|-----------|-----------------|
| `person.name` exists | `.personnel-card` (base) | Name at weight 600, full field stack |
| `person.name` absent | `.personnel-card` + `.personnel-name--anonymous` on name slot | Title in name position, italic, muted color |
| `person.isSelf === true` | `.personnel-card--self` | Left accent border + subtle background tint |
| `person.name` absent AND `person.isSelf` | Both modifiers apply | Italic muted name + accent border + tint |
