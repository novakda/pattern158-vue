### Severity Badge Color Contracts

Light mode badges use colored background with white text:
- **Critical:** background `--color-danger` (#c82333), text white, `--radius-pill`
- **High:** background `--color-accent-on-alt` (#7a5c00), text white, `--radius-pill`

Dark mode badges use colored text on transparent background with border:
- **Critical:** text #ffb3bd (existing dark override from Exhibit L), border 1px solid currentColor, background transparent
- **High:** text `--color-accent` (#d9b232), border 1px solid currentColor, background transparent

Source: Existing severity styling from `.page-exhibit-l .resolution-table` (lines 2735-2740, 229-234 of main.css).

Accent reserved for: Severity "High" badges only. Not used for interactive elements in this component (no links, no buttons).

## Component Visual Contract

### Desktop Table (viewport >= 769px)

```
+------------------------------------------------------------------+
| [Section Heading: "Findings" or custom]                          |
+------------------------------------------------------------------+
| Finding         | Description     | (optional cols)               |
|-----------------|-----------------|-------------------------------|
| Finding text    | Description...  | Severity badge / Background   |
| Finding text    | Description...  | Severity badge / Resolution   |
+------------------------------------------------------------------+
```

- Semantic `<table>` element, full width
- `border-collapse: collapse`
- Header row: `--color-background-alt` background, `--font-size-sm` at weight 600, 2px bottom border
- Body cells: `--space-sm` vertical / `--space-md` horizontal padding, 1px bottom border, `vertical-align: top`
- Row hover: `--color-background-alt` background transition
- Margin-top from heading: `--space-sm`

### Mobile Cards (viewport <= 768px)

```
+-----------------------------------+
| Finding: [finding text]           |
| Description: [description text]   |
| Severity: [badge]                 |
+-----------------------------------+
(gap: --space-md)
+-----------------------------------+
| Finding: [finding text]           |
| ...                               |
+-----------------------------------+
```

- Stacked card layout (not CSS grid columns — single column stack)
- Each card: `--space-md` padding, 1px `--color-border` border, `--radius-md` border-radius, `--color-surface` background
- Card gap: `--space-md`
- Field labels: `--font-size-sm` at weight 600, `--color-text-muted`
- Field values: `--font-size-sm` at weight 400, `--color-text`
- Field spacing within card: `--space-sm` margin-bottom per field

### Severity Badge (both modes)

- Display: inline element
- Border-radius: `--radius-pill` (20px)
- Padding: `--space-xs` vertical, `--space-sm` horizontal
- Font: `--font-size-xs`, weight 700, uppercase
- Light mode: colored background, white text
- Dark mode: colored text, transparent background, 1px border currentColor

### Column Detection Logic

Auto-inspect the `findings[]` array to determine which columns to show:

1. If ANY finding has `severity` populated: show Finding + Description + Severity (3-col)
2. If ANY finding has `background` populated: show Finding + Background + Resolution (3-col)
3. Otherwise: show Finding + Description (2-col)

Only one 3-col pattern applies per table (data does not mix patterns).

### Dual DOM Rendering

Both `<table>` and card grid are rendered in template. CSS media queries toggle visibility:

```css
.findings-table-desktop { display: table; }
.findings-table-mobile { display: none; }

@media (max-width: 768px) {
  .findings-table-desktop { display: none; }
  .findings-table-mobile { display: block; }
}
```
