    2. Create src/components/feedback/captureElement.ts as a pure function:
       ```typescript
       import { getCssSelector } from 'css-selector-generator'
       import type { ElementCapture } from './feedback.types'

       export function captureElement(el: HTMLElement): ElementCapture {
         const tag = el.tagName.toLowerCase()
         const selectorPath = getCssSelector(el, {
           blacklist: [/^data-v-/, /^data-test-/],
           maxCombinations: 100,
           maxCandidates: 100,
         })
         const boundingRect = el.getBoundingClientRect()
         const componentName = getVueComponentName(el)
         return { tag, selectorPath, boundingRect, screenshotDataUri: null, componentName }
       }
       ```

    3. Implement getVueComponentName as a private helper that walks up from `el` checking `(node as any).__vueParentComponent?.type?.name`. Walk up to 10 ancestors max. Return the first non-empty name found, or null.

    4. Create src/components/feedback/captureElement.test.ts with vitest tests covering the behaviors listed above. Mock css-selector-generator with vi.mock. Use a mock HTMLElement with getBoundingClientRect returning a stub DOMRect. For Vue component detection, attach `__vueParentComponent` to the mock element.

    5. The css-selector-generator options MUST exclude `data-v-*` and `data-test-*` via blacklist regexes per CONTEXT.md decision on stable selectors.
