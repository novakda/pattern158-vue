         const headings = doc.querySelectorAll('h1, h2, h3, h4, h5, h6')
         for (const oldEl of headings) {
           const currentLevel = Number(oldEl.tagName.charAt(1))

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Captured HTML → sanitizer | `rawHtml` originates from `page.locator('main#main-content').innerHTML()` in Phase 48. ## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-49-01 | Tampering | sanitizeHtml — HTML injection into output markdown | mitigate | Turndown's default rules escape raw HTML fragments; this plan's sanitizer removes `<script>/<style>/<noscript>` BEFORE Turndown runs, so any injected `<script>` can never reach the Markdown output. Block-level element escaping is Turndown's responsibility downstream; we guarantee no script-carrying subtrees survive sanitization. |
| T-49-02 | Denial of Service | sanitizeHtml — pathological HTML (deep nesting, many elements) | accept | domino + happy-dom both have O(n) parse complexity. The largest realistic input is the FAQ page main-content HTML (~200KB, ~5K elements). `querySelectorAll('*')` is O(n), attribute iteration is amortized O(1) per attr. No regex on the full HTML string. Accepting DoS risk: editorial capture is a dev-local tool, not an exposed service. |
| T-49-03 | Tampering | demoteHeadings — prototype pollution via crafted heading content | mitigate | `oldEl.tagName.charAt(1)` returns a string char. `Number()` on non-digit char returns NaN; `Math.min(6, NaN + 2) === NaN`, which fails the `newLevel === currentLevel` guard loosely but `newLevel = NaN` passed to `createElement(`h${NaN}`)` throws a DOM error, surfacing malformed input as a loud failure rather than silent corruption. Acceptable — headings come from the trusted live site; malformed input is a capture-phase bug that should fail loudly. |
| T-49-04 | Information Disclosure | data-v-* attribute leakage in output | mitigate | Primary purpose of the attribute walk. Every element's attributes are enumerated and any name matching `^data-v-/` is removed. Test coverage in Plan 49-04 asserts this across multiple scope IDs. |
| T-49-05 | Information Disclosure | aria-hidden subtree leakage (hidden icons, decorative content) | mitigate | Subtree strip removes every `[aria-hidden="true"]` element and all children. Editorial corpus never contains the "visually hidden" content a screen reader would skip. |
