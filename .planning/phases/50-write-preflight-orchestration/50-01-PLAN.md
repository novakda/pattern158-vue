<objective>
Create the new module `scripts/editorial/document.ts` that turns a `DocumentAssemblyInput` (converted pages + failures + run provenance) into a deterministic monolithic Markdown string. The module is a pure, side-effect-free function of its inputs — no wall-clock reads, no I/O, no Playwright. Phase 50-03's `index.ts` orchestrator calls `assembleDocument(input)` once per run and hands the resulting string to the writer.

Public surface (exactly 6 functions + 2 interfaces):

```ts
export interface RouteFailure {
  readonly route: Route
  readonly error: unknown
  readonly httpStatus?: number
}

export interface DocumentAssemblyInput {
  readonly config: EditorialConfig
  readonly captured: readonly ConvertedPage[]
  readonly failures: readonly RouteFailure[]
  readonly capturedAt: string           // ISO 8601 UTC
  readonly toolVersion: string           // e.g. "editorial-capture/abc1234+dirty"
  readonly siteVersionSha: string        // parsed from home route meta; "" fallback
}

export function assembleDocument(input: DocumentAssemblyInput): string
export function buildFrontmatter(input: DocumentAssemblyInput): string
export function buildToc(captured: readonly ConvertedPage[], failures: readonly RouteFailure[]): string
export function buildRouteSection(page: ConvertedPage, capturedAt: string): string
export function buildFailedRouteSection(failure: RouteFailure, capturedAt: string): string
export function slugForRoute(route: Route): string
```

Plus a companion Vitest suite in `scripts/editorial/__tests__/document.test.ts` that covers every export with inline fixtures (hermetic — no fs, no network, no prior phase modules other than type imports).

**Purpose.** SHAP-01..06 are all document-shape requirements: single monolithic file, top-level YAML frontmatter with provenance, auto-generated ToC with github-slugger anchors, `## Route: /path` per-route headings, per-page metadata blockquotes, `---` separators between routes, H1-to-H3 demotion delegated to Phase 49. This plan delivers every one of those behaviors as pure functions with tests.

**Output.** A ready-to-call assembler that Phase 50-03 composes into its pipeline. Zero business logic about capture, convert, write, or orchestration lives here — every step is a deterministic function of `DocumentAssemblyInput`.

       export function buildToc(
         routes: readonly Route[],
         captured: readonly ConvertedPage[],
         failures: readonly RouteFailure[],
       ): string {
         const lines: string[] = ['## Contents', '']
         const capturedPaths = new Set(captured.map((p) => p.route.path))
         const failedPaths = new Set(failures.map((f) => f.route.path))
         for (const route of routes) {
           const slug = slugForRoute(route)
           if (capturedPaths.has(route.path)) {
             lines.push(`- [${route.label}](#${slug})`)
           } else if (failedPaths.has(route.path)) {
             lines.push(`- [${route.label} (failed)](#${slug})`)
           }
           // routes not in either set are silently skipped (should never happen in production).
         }
         return lines.join('\n')
       }
