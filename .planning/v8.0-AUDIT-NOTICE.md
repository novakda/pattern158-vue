## Summary

v8.0 built a Playwright-based live-site editorial capture tool (`scripts/editorial/`) that drives headless Chromium through production `pattern158.solutions`, converts every rendered page to deterministic Markdown via Turndown + GFM, and assembles a single monolithic review document at `<vault>/career/website/site-editorial-capture.md` with per-route screenshots, provenance frontmatter, auto-generated ToC, and structured per-page metadata blocks.

The tool is shipped, fully tested (401 unit tests across 24 test files, all green), and validated end-to-end against live production — a full run captures all 22 non-excluded routes in ~42s with zero failures, producing a ~186 KB Markdown artifact + 22 full-page screenshots. Atomic writes + PID-temp + `fsp.rename` make re-runs idempotent; the scaffold for Dan's editorial findings auto-emits on the first run.

The **editorial review phase** (Phase 51, EDIT-01..04) is the intentionally manual portion of the milestone: Dan reads the captured artifact, populates `<vault>/career/website/site-editorial-findings.md` (scaffold already emitted), and produces prioritized findings cross-referenced against the three career positioning documents.

The **v9.0 direction verdict** (AUDT-02, AUDT-03) was locked 2026-04-21 via a dedicated smart-discuss session that examined the v9.0 candidate space independently of the editorial findings pathway. The chosen direction preserves the Vue site as-is (no rewrite, no framework migration) and adds a new TiddlyWiki generation layer fed by the v8.0 editorial-capture pipeline. Editorial findings (EDIT-01..04) remain open as documented async human work — they are not a gate on v9.0 planning under the locked direction, because the direction does not modify the Vue site itself.

## What was built

### Phase 46 — Scaffold
`scripts/editorial/` directory, separate `tsconfig.editorial.json` project reference, Vitest `scripts` project extended to include the new suite, `pnpm editorial:capture` script, Playwright 1.59.1 + Turndown 7.2.4 + `@joplin/turndown-plugin-gfm` 1.0.64 + `@types/turndown` + `happy-dom` + `yaml` + `github-slugger` devDeps installed, SCAF-08 forbidden-pattern discipline documented and grep-gated per phase.

### Phase 47 — Config + Routes (pure logic)
`config.ts` with hand-rolled CLI arg parser (`--output`, `--base-url`, `--headful`, `--mirror`), env fallback (`EDITORIAL_OUT_PATH`, `EDITORIAL_BASE_URL`), typed `ConfigError` (exit code 2), preflight path validation (parent exists + writable + valid `https:` URL). `routes.ts` with deterministic ordered `Route[]` from static route list + `exhibits.json` via `fs.readFile` + `JSON.parse` (no `@/` aliases, no ESM JSON import assertions), excluded-set (`/review`, `/diag/*`, `/portfolio`, `/testimonials`, 404 fallback). 49 new unit tests locked this phase.

### Phase 48 — Capture (Playwright IO)
`capture.ts` — headless Chromium driver with:
- Single-browser / single-context / fresh-page-per-route lifecycle; 1500ms deterministic inter-request delay; 30s goto + 10s selector timeouts; no retries (abort-on-first-error within strict `captureRoutes`).
- Cloudflare cache-buster `?_cb=<route-slug>` + `Cache-Control: no-cache` + layered 3-signal interstitial detection (title + body size + challenge-platform markup) with `CaptureError` abort on trip.
- `<main id="main-content">` scoping so NavBar/FooterBar/skip-link never leak into captures.
- FAQ pre-capture hooks (CRIT-01/02 mitigation): filter-all click first, then atomic in-page JS click of every `[aria-expanded="false"]` trigger (bug found and fixed via live smoke — stale Locator handles on DOM mutation), hard-fail if rendered count ≠ `faq.json` length.
- Silent-SPA-404 detection via `.exhibit-detail-title` assertion on exhibit routes.
- Per-route full-page PNG screenshots at 1280×800 light theme; per-route console errors and SEO meta (title + `meta[name="description"]`) captured into `CapturedPage`; `cf-cache-status` response header preserved.

Shipped with 36 hermetic unit tests using `vi.spyOn(chromium, 'launch')` for integration coverage.

### Phase 49 — Convert (Turndown)
`convert.ts` — deterministic HTML → Markdown pipeline with:
- Pre-Turndown sanitization via `happy-dom`: strip `<script>`/`<style>`/`<noscript>`/`[aria-hidden="true"]` subtrees; strip every `data-v-*` SFC attribute; demote every h1-h6 by 2 levels (clamped at h6) so page H1 renders as H3 under the per-route `## Route:` heading.
- Turndown 7.2.4 + full `@joplin/turndown-plugin-gfm` plugin (tables + strikethrough + task lists). Ambient `.d.ts` shim added because upstream ships no types.
- Custom `image-alt-only` rule (emit alt text only, skip image entirely if no alt — no base64, no `![]()`) registered BEFORE the GFM plugin.
- Custom `pattern158-badges` rule (class allowlist `badge|badge-\w+|pill|chip|tag|tag-\w+|severity-\w+|category-\w+` rendered as `**bold**`, icon+text flatten to text-only).
- `collapseBlankLines` post-Turndown normalization (`/\n{3,}/g → \n\n`, linear-time, idempotent).
- Extended `ConvertedPage` shape with `consoleErrors`, `screenshotPath`, `cfCacheStatus?` carried through from capture.

Shipped with 44 hermetic test cases including explicit byte-equal determinism self-test (CONV-02). Code review surfaced + fixed: `demoteHeadings` attribute preservation, `sanitizeHtml` parse simplification, dead `linkReferenceStyle` config, badge-regex negative tests.

### Phase 50 — Write + Preflight + Orchestration
Three modules wired end-to-end:
- `document.ts` — pure document assembler: `buildFrontmatter` via `yaml.stringify` (custom minimal builder, NOT the v7.0 Obsidian-note serializer — doesn't generalize); `buildToc(routes, captured, failures)` via `github-slugger` with `route-<slug>` anchors matching `## Route:` heading slugs; `buildRouteSection` with blockquote metadata (`Captured`, `Status`, `CF Cache`, `Title`, `Description` — omit-empty); `buildFailedRouteSection` placeholder; `slugForRoute` deterministic path-to-slug.
- `write.ts` — `atomicWrite` (temp + PID suffix + `fsp.rename`, UTF-8 explicit, `\n` only, best-effort temp cleanup on failure); `writePrimaryAndMirror` for optional `.planning/research/` byte-identical dual-write; `emitFindingsScaffold` (Phase 51 EDIT-05) idempotent non-overwriting scaffold emission.
- `index.ts` — orchestrator that drives `capturePage` directly in its own per-route loop (bypassing strict `captureRoutes`), re-throws Cloudflare interstitial but logs-and-continues on any other `CaptureError`, reads wall-clock exactly once at run start, emits stdout + stderr-JSON summary, exit code 0 iff all routes httpStatus==200 AND mainHtml≥200 bytes AND zero failures.

Shipped with 69 new tests covering document assembly determinism, atomic write idempotency, exit-code preconditions, interstitial abort, per-route failure continuation, CLI-invocation guard via `fileURLToPath`. Code review surfaced + fixed: browser+context cleanup hardening, ESM path-compare robustness.

### Candidate directions — verdict

| # | Direction | Go/No-Go | Rationale |

Rebuilding the Vue site in another framework replaces the Vue exhibit rather than adding to the collection; the whole point of Rosetta Stone is that each framework implementation stands alongside the others as parallel evidence. |
| 4 | **Other / hybrid — tzk-style living TiddlyWiki fed by live-site capture** | **GO** | v9.0 scope. Vue site continues unchanged as canonical source of truth; v8.0 editorial-capture pipeline is reused as the HTML intake; new DOM extractors decompose captured HTML into atomic tiddlers (personnel, findings, technologies, testimonials); Pattern 158 brand theme + tzk-style private/public publishing workflow produce a deployable living wiki. Framework-agnostic by construction — tiddler files are portable data, not framework code. |

### Rosetta Stone alignment check

The chosen direction is **maximally Rosetta-Stone-aligned** among the four candidates:

- **Preserves the Vue exhibit** — Vue site continues unchanged; it remains a distinct portfolio artifact with its own architecture story.
- **Adds a new artifact class (TiddlyWiki)** alongside Vue without replacing it — the portfolio gains breadth (a second, fundamentally different front-end paradigm: reactive SPA ↔ single-file wiki) without losing the Vue evidence.
- **Canonical-source-is-the-live-site** means any future Rosetta Stone rebuild (Svelte, Astro, 11ty, etc.) can plug into the same editorial-capture + DOM-extractor pipeline — the intake layer is framework-agnostic, so each new framework exhibit inherits the data layer instead of reinventing it.
- **No foreclosure** — choosing to build the TiddlyWiki layer does not commit the portfolio to TiddlyWiki as the sole wiki/documentation surface; it adds one more concrete Rosetta implementation with clean separation between capture (upstream) and rendering (downstream).
