## Why Aborted

The real need is **editorial, not publication.** Dan's immediate goal is to read the full rendered site as a single semantic document, identify inconsistencies with recent career positioning work, make structural and copy decisions, then rebuild.

v7.0's source-module extraction pipeline was well-engineered for its original goal (derived renderings from `src/content/*.ts` with type preservation and drift detection) but misses the fidelity editorial work requires:

1. **Incomplete coverage** — hardcoded component text, data modules rendered into prose, exhibit detail pages only materialize at render time
2. **No reading-order fidelity** — source modules are unordered; page templates define the order readers encounter
3. **No composition fidelity** — component composition, conditional rendering, dynamic data interpolation only exist in rendered output
4. **Direction lock-in** — v7.0 treats Vue as canonical source; the longer-term Rosetta Stone vision inverts that

Full reasoning in `.planning/v7.0-ABORT-NOTICE.md`.
