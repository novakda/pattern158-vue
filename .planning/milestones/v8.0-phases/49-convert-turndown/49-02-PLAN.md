<objective>
Implement `configureTurndown(): TurndownService` in `scripts/editorial/convert.ts` AND create a local TypeScript ambient module shim at `scripts/editorial/turndown-plugin-gfm.d.ts` for the untyped `@joplin/turndown-plugin-gfm` package.

Factory returns a fresh configured TurndownService instance:
1. Locked service config per CONTEXT.md (atx headings, `-` bullets, fenced code blocks, `*` em, `**` strong, inlined links).
2. Custom `img` rule: emit alt text only when alt is a non-empty string; skip image entirely otherwise. Never emit `![alt](src)` Markdown. Never emit base64 data URLs.
3. Custom `pattern158-badges` rule: detect `<span>` or `<a>` elements with classes matching the site's badge design-system (`.badge`, `.badge-*`, `.pill`, `.chip`, `.tag`, `.tag-*`, `.severity-*`, `.category-*`); emit their text content wrapped in `**bold**`. Nested icon subtrees inside a badge are flattened to text-only via the rule's `replacement` handler.
4. GFM plugin: `service.use(gfm)` AFTER custom rules register, so the GFM plugin's own strikethrough/tables/task-list rules don't shadow ours.

**Purpose:** CONV-01 (Turndown + full GFM plugin), CONV-03 (images alt-text only), CONV-05 (badge passthrough as bold — class allowlist explicitly includes `badge-\w+` variants per checker warning #4), CONV-06 (DOM-order preservation — default Turndown behavior, verified by Plan 49-04 tests), CONV-07 (href verbatim — default Turndown behavior with `inlined` link style).

// node_modules/turndown/types/turndown.d.ts
declare class TurndownService {
  constructor(options?: TurndownService.Options)
  addRule(key: string, rule: TurndownService.Rule): this
  use(plugin: TurndownService.Plugin | TurndownService.Plugin[]): this
  turndown(input: string | TurndownService.Node): string
}

namespace TurndownService {
  interface Options {
    headingStyle?: 'setext' | 'atx'
    bulletListMarker?: '-' | '+' | '*'
    codeBlockStyle?: 'indented' | 'fenced'
    emDelimiter?: '_' | '*'
    strongDelimiter?: '__' | '**'
    linkStyle?: 'inlined' | 'referenced'
    linkReferenceStyle?: 'full' | 'collapsed' | 'shortcut'
    // ... plus others not used by this plan
  }

  interface Rule {
    filter: string | string[] | FilterFunction
    replacement: ReplacementFunction
  }

  type FilterFunction = (node: Node, options: Options) => boolean
  type ReplacementFunction = (content: string, node: Node, options: Options) => string

  type Plugin = (service: TurndownService) => void
}

### Badge class matcher regex (REVISED per checker warning #4)

Checker required `badge-\w+` alternation in the filter regex. The site inventory (CONTEXT.md line 140) explicitly lists `.badge` AND `.badge-*` as distinct classes (e.g., `<span class="badge-high">HIGH</span>` — without a co-occurring `.badge` parent class). Without `badge-\w+` in the alternation, `class="badge-high"` alone would not match the filter and would not render as `**bold**`.

**Revised regex (MANDATORY):**
```typescript
/(^|\s)(badge|badge-\w+|pill|chip|tag|tag-\w+|severity-\w+|category-\w+)(\s|$)/
```

Alternation list in order: `badge` | `badge-\w+` | `pill` | `chip` | `tag` | `tag-\w+` | `severity-\w+` | `category-\w+`.

Note: regex still has only bounded `\w+` quantifiers (no nested quantifiers, no backtracking explosion) — remains linear-time / ReDoS-safe. Update the threat-model mitigation text accordingly (T-49-06 still applies with the revised regex).

Test coverage (added to Plan 49-04): `<span class="badge-high">HIGH</span>` → `**HIGH**` must pass.

       ```ts
       /**
        * configureTurndown — factory for a configured TurndownService. Returns a fresh
        * instance each call (callers own the lifecycle; GFM plugin registration is
        * idempotent per-instance). Custom rules register BEFORE .use(gfm) so the
        * GFM plugin's own rules don't shadow ours.
        *
        * CONV-01: Turndown 7.2.4 + @joplin/turndown-plugin-gfm full plugin.
        * CONV-03: images emit alt text only — no ![]() Markdown, no base64 data URLs.
        * CONV-05: badge/badge-*/pill/chip/tag/tag-*/severity-*/category-* spans render as **bold**.
        * CONV-06: DOM-order preservation is Turndown's default — verified by tests.
        * CONV-07: hrefs preserved verbatim via linkStyle: 'inlined'.
        */
       export function configureTurndown(): TurndownService {
         const service = new TurndownService({
           headingStyle: 'atx',
           bulletListMarker: '-',
           codeBlockStyle: 'fenced',
           emDelimiter: '*',
           strongDelimiter: '**',
           linkStyle: 'inlined',
           linkReferenceStyle: 'full',
         })

         // CONV-03: alt-text-only image rule. Override Turndown's default image
         // handler so ![]( ) Markdown and base64 data URLs never appear in output.
         service.addRule('image-alt-only', {
           filter: 'img',
           replacement: (_content, node) => {
             const alt = (node as Element).getAttribute('alt') ?? ''
             return alt.length === 0 ? '' : alt
           },
         })

         // CONV-05: pattern158 design-system badge/pill passthrough as **bold**.
         // Class allowlist matches Phase 5+/7+ design tokens: badge, badge-*, pill,
         // chip, tag, tag-*, severity-*, category-*. The `badge-\w+` alternation
         // catches compound variants like `badge-high` that appear standalone
         // (without a co-occurring `.badge` base class). Nested icon subtrees
         // flatten to text-only via Turndown's natural content recursion.
         service.addRule('pattern158-badges', {
           filter: (node) => {
             const tag = node.nodeName.toLowerCase()
             if (tag !== 'span' && tag !== 'a') return false
             const cls = (node as Element).getAttribute('class') ?? ''
             return /(^|\s)(badge|badge-\w+|pill|chip|tag|tag-\w+|severity-\w+|category-\w+)(\s|$)/.test(cls)
           },
           replacement: (content) => {
             const trimmed = content.trim()
             return trimmed.length === 0 ? '' : `**${trimmed}**`
           },
         })

         // CONV-01: GFM plugin AFTER custom rules — so tables/strikethrough/task-list
         // rules from the plugin don't shadow the img + badge rules above.
         service.use(gfm)

         return service
       }
       ```
