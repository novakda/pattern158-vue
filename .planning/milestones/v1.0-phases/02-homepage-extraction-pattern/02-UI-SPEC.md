## Typography

All font sizes use CSS custom properties. Four roles are declared for this phase:

| Role | CSS Var | Size | Weight | Line Height |
|------|---------|------|--------|-------------|
| Body | `--font-size-base` | 1rem (16px) | 400 (regular) | 1.6 (`--line-height-base`) |
| Label / Meta | `--font-size-sm` | 0.875rem (14px) | 400 (regular) | 1.5 (`--line-height-sm`) |
| Heading (h2/h3) | `--font-size-xl` | 1.5rem (24px) | 400 — Bebas Neue renders visually bold by default | 1.5 (`--line-height-xl`) |
| Display (h1 hero) | `--font-size-4xl` | 3.5rem (56px) | 400 — Bebas Neue | 1.2 (`--line-height-4xl`) |

Weight contract: exactly 2 weights — 400 (all body and heading text; Bebas Neue renders thick at 400) and 600 (semibold for stat numbers and finding labels where emphasis is needed within body copy).

Source: `src/assets/css/main.css` — `--font-size-*` and `--line-height-*` token blocks.

## Color

All values are CSS custom properties. Light mode tokens listed; dark mode overrides are already in `main.css` via `[data-theme="dark"]`. All token pairs are WCAG AA verified (source: inline comments in `main.css`).

| Role | CSS Var | Light Value | Dark Value | Usage |
|------|---------|-------------|------------|-------|
| Dominant (60%) | `--color-background` | `#faf9f6` | `#1a2838` | Page background, primary surface |
| Secondary (30%) | `--color-surface` / `--color-background-alt` | `#ffffff` / `#e8e8e8` | `#2a3f54` / `#243447` | Cards (FindingCard, SpecialtyCard), alternating sections |
| Accent (10%) | `--color-primary` | `#0e7c8c` | `#30c9dc` | Links, `.btn-primary` background, `.finding-label` background, `TechTags` / tech pills border |
| Semantic: muted text | `--color-text-muted` | `#666666` | `#ababab` | `.finding-meta`, `.section-subtitle`, `StatItem` labels |
| Semantic: heading | `--color-heading` | `#1a2838` | `#e8e8e8` | All `h1`–`h4` heading text |
| Semantic: body | `--color-text` | `#2d3436` | `#e8e8e8` | Body paragraph text |
| Semantic: border | `--color-border` | `#dddddd` | `#5d7289` | FindingCard border, SpecialtyCard border |
| Semantic: danger | `--color-danger` | `#c82333` | `#ff8c99` | Not used in Phase 2 (no destructive actions) |

Accent (`--color-primary`) is reserved for: internal `<router-link>` text, `.btn-primary` background fill, `.finding-label` badge background, tech pill / `TechTags` border, and focus ring shadow (`--shadow-focus`). It is NOT used for decorative borders, backgrounds of card surfaces, or heading text.

Source: `src/assets/css/main.css` — color token block and `[data-theme="dark"]` overrides.

## Copywriting Contract

This phase ports content that already exists verbatim in `src/pages/HomePage.vue` (source of truth: live 11ty site). No new copy is authored. The contract below documents the specific strings to preserve exactly.

| Element | Copy |
|---------|------|
| Primary CTA (hero) | "Get in Touch" — maps to `<router-link to="/contact">` |
| Secondary CTA (hero) | "View My Work" — maps to `<router-link to="/portfolio">` |
| Field Reports CTA | "View All Field Reports" — maps to `<router-link to="/testimonials">` |
| Philosophy link | "Full details →" — maps to `<router-link to="/philosophy#influences">` |
| Philosophy repeat link | "Full philosophy →" — maps to `<router-link to="/philosophy#influences">` |
| Empty state heading | Not applicable — no dynamic data that could be absent |
| Empty state body | Not applicable — all content is static |
| Error state | Not applicable — no async operations, no form submissions |
| Destructive confirmation | None — no destructive actions in Phase 2 |

HTML entities in source copy must be preserved as Vue template equivalents or literal Unicode characters:
- `&mdash;` → `—`
- `&hellip;` → `…`
- `&rarr;` → `→`
- `&ldquo;` / `&rdquo;` → `"` / `"`
- `&rsquo;` → `'`
- `&lsquo;` → `'`

Source: `src/pages/HomePage.vue` existing content.

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| All internal links | `<router-link :to="...">` — zero raw `.html` hrefs (phase success criterion) |
| Exhibit links | `<router-link to="/exhibits/exhibit-e">` clean paths — no `.html` extension — will render NotFoundPage until exhibit pages are ported |
| Hash anchor links | `<router-link to="/philosophy#influences">` — Vue Router handles natively |
| External links | Not present on HomePage — no `target="_blank"` needed |
| Theme toggle | Handled by existing `ThemeToggle.vue` and `[data-theme="dark"]` CSS — no new interaction |
| Focus management | All interactive elements receive `:focus-visible` ring via `--shadow-focus` — inherit from existing global styles |

## Responsive Breakpoints

Three breakpoints are required for Phase 2 success criteria (ROADMAP.md Phase 2 criterion 1):

| Breakpoint | Width | Layout expectations |
|------------|-------|---------------------|
| Mobile | 375px | Single column; stats row wraps to 2×2 grid; specialties stack; hero name at reduced display size |
| Tablet | 768px | Specialties move to 2-column grid; hero gets more breathing room |
| Desktop | 1280px | Full layout; specialties at 4-column; stats row single row |

Both light and dark themes must pass visual parity check at all three breakpoints (Phase 2 success criterion 1).
