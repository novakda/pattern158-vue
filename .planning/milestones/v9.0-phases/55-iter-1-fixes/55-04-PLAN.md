Key nuance: `emitPersonTiddlers` takes a single `client` per call (ATOM-01 locked tag format `[[{Client}]]`). Since different exhibits have different clients, the call site must group personnel by client and invoke once per client. Wave-1 extract-all bundle has `personnelByExhibit` as a flat array; each entry's `sourceExhibitLabel` resolves to an `Exhibit` in `bundle.exhibits`, whose `.client` field is the client string.

Grouping strategy for person tiddlers:
```typescript
// Build label → client lookup
const clientByLabel = new Map<string, string>()
for (const ex of bundle.exhibits) clientByLabel.set(ex.label, ex.client)

// Group personnel entries by client
const personnelByClient = new Map<string, PersonnelEntry[]>()
for (const p of bundle.personnelByExhibit) {
  const client = clientByLabel.get(p.sourceExhibitLabel)
  if (client === undefined) continue  // skip orphans (no matching exhibit)
  const bucket = personnelByClient.get(client)
  if (bucket === undefined) personnelByClient.set(client, [p])
  else bucket.push(p)
}

// Emit per-client
const personTiddlers: Tiddler[] = []
for (const client of Array.from(personnelByClient.keys()).sort()) {
  personTiddlers.push(...emitPersonTiddlers(personnelByClient.get(client)!, { client }))
}
```

The sorted client iteration ensures deterministic output order across runs (mirrors Phase 54's deterministic-output policy).

Finding / Technology / Testimonial generators take a single flat array — one invocation each on `bundle.findingsByExhibit`, `bundle.technologiesByExhibit`, `bundle.testimonials`.
