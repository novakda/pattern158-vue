## Spacing Scale

Declared values from existing `--space-*` tokens (rem-based, not 8pt grid):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 0.25rem (4px) | Inline gaps, label margins |
| sm | 0.5rem (8px) | Compact element spacing |
| ms | 0.75rem (12px) | Mobile horizontal padding |
| md | 1rem (16px) | Default card padding, card gap |
| lg | 1.5rem (24px) | Section padding |
| xl | 2rem (32px) | Layout gaps |
| 2xl | 3rem (48px) | Major section breaks |

Exceptions: Group entry cards use reduced padding `var(--space-sm) var(--space-md)` (8px vertical, 16px horizontal) instead of the standard `var(--space-md)` (16px all sides) to convey compact/subordinate treatment.

### Card Heading Logic (CARD-03)

Priority cascade for the first cell (card heading):

1. `p.name` if present and non-empty -- render as card heading
2. `p.title` if present and non-empty (and no name) -- render as card heading
3. `p.role` if present and non-empty (and no name/title) -- render as card heading

The heading field gets: `font-weight: 700`, `font-size: var(--font-size-lg)`, `font-family: var(--font-heading)`, `color: var(--color-heading)`, `line-height: 1.3`.

## Component Behavior Contract

### Mobile Cards (breakpoint at existing media query)

**Individual entry (default):**
- Background: `var(--color-background-alt)`
- Border-radius: `var(--radius-md, 8px)`
- Padding: `var(--space-md)`
- No left border (matches v5.1 card pattern)
- First field = heading treatment per cascade above
- Subsequent fields = `var(--font-size-sm)`, `var(--color-text-medium)`

**Group entry (CARD-01):**
- CSS class: `.personnel-entry-group` on `<tr>`
- Opacity: `0.7` on the entire card
- Left border: `4px dashed var(--color-border)`
- Padding: `var(--space-sm) var(--space-md)` (reduced vertical)
- Before the heading text, inject "GROUP" label via CSS pseudo-element or inline `<span>`
- Group label styling: `font-size: var(--font-size-xs)`, `font-weight: 600`, `text-transform: uppercase`, `letter-spacing: 0.5px`, `color: var(--color-text-muted)`, `display: block`, `margin-bottom: var(--space-xs)`

**Anonymized entry (CARD-02):**
- CSS class: `.personnel-entry-anonymized` on `<tr>`
- All text within card: `font-style: italic`
- Heading text color: `var(--color-text-muted)` instead of `var(--color-heading)`
- No explicit label — italic treatment is the visual distinction

### Desktop Table Rows (CARD-04)

**Individual entry (default):**
- Standard row styling (no changes from existing)

**Group entry:**
- CSS class: `.personnel-entry-group` on `<tr>`
- Text color: `var(--color-text-muted)` on all `<td>` in the row
- `font-style: italic` on all `<td>` in the row

**Anonymized entry:**
- CSS class: `.personnel-entry-anonymized` on `<tr>`
- `font-style: italic` on name and title `<td>` cells

### Template Changes

**Dead code removal:**
- Remove the `v-if="p.involvement"` template branch (lines 54-57 thead, lines 67-70 tbody in InvestigationReportLayout.vue)
- Exhibit L was normalized in Phase 28 — involvement field no longer exists in data

**entryType-driven class binding:**
- `<tr>` gets dynamic class: `:class="{ 'personnel-entry-group': p.entryType === 'group', 'personnel-entry-anonymized': p.entryType === 'anonymized' }"`

**Heading field logic:**
- First `<td>` renders the best available field: `{{ p.name || p.title || p.role }}`
- `data-label` attribute matches the source field: `"Name"` if name present, `"Title"` if title used as fallback, `"Role"` if role used as fallback
