## Phase Details

### Phase 30: Cross-Page Content Audit
**Goal**: User can review a comprehensive audit document showing every FAQ answer compared against authoritative content on other site pages
**Depends on**: Nothing (first phase of v5.3)
**Requirements**: AUDIT-01, AUDIT-02
**Success Criteria** (what must be TRUE):
  1. Every FAQ question/answer has been compared against relevant content on HomePage, TechnologiesPage, PhilosophyPage, ContactPage, CaseFilesPage, and exhibit data
  2. Each audit finding is classified by issue type (stale reference, factual drift, content overlap, missing content)
  3. Each finding includes a recommended fix or resolution action
**Plans:** 1/1 plans complete
Plans:
- [x] 30-01-PLAN.md — Cross-page content audit and validation

### Phase 31: Content Fixes
**Goal**: FAQ answers are factually accurate and use current site terminology
**Depends on**: Phase 30
**Requirements**: REFS-01, REFS-02, ACCY-01, ACCY-02, ACCY-03, ACCY-04
**Success Criteria** (what must be TRUE):
  1. FAQ answers reference "case files" instead of "portfolio" wherever the site section is mentioned
  2. FAQ technology listing matches current primary technologies shown on TechnologiesPage (including React, Python, Power Platform, Claude Code)
  3. FAQ industry list is consistent with client references across Contact and Case Files pages
  4. FAQ AI/automation answer reflects current tooling (Claude Code as primary, Copilot Studio, practical AI focus) and "GitHub Spec Kit" reference is verified or corrected
  5. Accessibility standard phrasing is consistent between FAQ and Contact page ("WCAG 2.1 AA" / "Section 508")
**Plans:** 1/1 plans complete
Plans:
- [x] 31-01-PLAN.md — Fix stale references, tech listing, industry list, accessibility phrasing, and internal duplication

### Phase 32: Overlap Resolution
**Goal**: FAQ content complements rather than duplicates Contact and Philosophy pages, with each page serving a distinct purpose
**Depends on**: Phase 31
**Requirements**: OVLP-01, OVLP-02, OVLP-03
**Success Criteria** (what must be TRUE):
  1. FAQ work arrangement, communication, and independence questions provide value beyond what Contact page CultureFit/RoleFit sections already cover
  2. FAQ "typical workflow" answer is differentiated from Philosophy page "How I Work" section content
  3. Where overlap exists, FAQ adds a distinct angle (e.g., quick-scan summary vs. detailed philosophy, different audience framing, or consolidated reference)
