## Key Decisions (Milestone-Level)

1. **Canonical source = live pattern158.solutions** via Playwright DOM + screenshots (NOT JSON files, NOT Obsidian).
2. **Atomic decomposition:** personnel + findings + technologies + testimonials all get per-entity tiddlers (~250 atomic tiddlers in the 367-tiddler corpus).
3. **Phase boundaries LOCKED** between extractors (pure DOM parse) → generators (pure data transforms) → wiring (generate.ts composes). No cross-boundary modifications.
4. **happy-dom everywhere** for HTML parsing.
5. **SCAF-08 discipline** (no wall-clock, no parallel iteration, deterministic) enforced via grep-based acceptance gates on every source + test file.
6. **No JSDoc** convention maintained throughout (CONVENTIONS.md).
7. **Exhibit tiddler title canonicalized to short form** (`"Exhibit A"`) during Phase 55 hotfix — resolved 267 orphaned cross-links, enables integrity gate.

## v9.0 Retrospective Notes

**What worked:**
- Phase-53/54 boundary discipline (extractors pure, generators pure) made Wave 2 parallelism tractable.
- TDD RED→GREEN in every plan caught broken code before commit.
- SCAF-08 determinism made the integrity gate possible — byte-identical regeneration means `git diff` is signal, not noise.
- The HARD GATE in Phase 55 caught the "Exhibit Exhibit A" double-prefix bug that would have silently shipped as orphaned links otherwise.

**What we'd do differently:**
- Use `branching_strategy: "worktree"` for Wave-2 parallel execution to avoid commit-attribution scrambling on shared main.
- Catch the title-format mismatch (extractor label vs. generator cross-link target vs. iter-1 tiddler title) at plan-phase time by threading a "canonical title" spec through Phase 53's CONTEXT.md.
